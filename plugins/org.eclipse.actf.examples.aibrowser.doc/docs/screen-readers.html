Using with Screen Readers

aiBrowser detects screen readers automatically when aiBrowser is launched, so the screen reader which you usually used should be running before aiBrowser is launched. aiBrowser supports JAWS for Windows and Window-Eyes.

  • Using with JAWS for Windows
  • Using with Window-Eyes

Using with JAWS for Windows

Settings

Voice Setting
aiBrowser automatically changes the voice settings to JAWS if JAWS is detected when aiBrowser is launched. If JAWS doesn't speak then the voice settings should be changed.

Differences between using aiBrowser with JAWS and without JAWS

Appearance
The appearance of aiBrowser when running with JAWS is slightly different from that when running without JAWS. The tree view, the audio description view, and the address text form are hidden when running with JAWS, to avoid double readings by aiBrowser and JAWS.
Speak All function (Alt + Down Arrow)
While executing the "speak all" function, only text read by aiBrowser is shown instead of text in the browser.

Using with Window-Eyes

Settings

Voice Setting
aiBrowser automatically changes the voice settings to Window-Eyes if Window-Eyes is detected when aiBrowser is launched. If Window-Eyes doesn't speak then the voice settings should be changed.

Differences between using aiBrowser with Window-Eyes and without Window-Eyes

Speak All function (Alt + Down Arrow)
This function doesn't work with Window-Eyes setting.
